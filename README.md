Insurance & Reinsurance Policy and Claims Management System

Features
1. User accesses the application URL 
2. Login screen is displayed 
3. User enters credentials 
4. System authenticates user and retrieves role 
5. User is redirected to role-specific dashboard 
6. Navigation menu is rendered based on permissions
7. 
1. User navigates to Policy List screen 
2. Existing policies are displayed in a table 
3. User clicks 'Create Policy' 
4. Multi-step Create Policy Wizard is launched 
5. User completes General, Coverage, and Review steps 
6. Policy is saved in DRAFT or submitted for approval 
7. Policy Details screen displays current status and actions

8. 4.1 Journey: Policy Creation & Approval 
1. Underwriter logs into the system 
2. Creates a policy in DRAFT status 
3. Submits policy for approval 

4. System calculates exposure and retention 
5. Policy transitions to ACTIVE status 
6. Audit log is generated

3. User Roles 
Underwriter – Creates and approves policies 
Claims Adjuster – Reviews and settles claims 
Reinsurance Manager – Manages treaties and validates allocations 
Admin – Manages users, roles, and system configuration
